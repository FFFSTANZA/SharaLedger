1. BRAND IDENTITY & VISUAL LANGUAGE
================================================================================

BRAND PERSONALITY:

PRIMARY TRAITS:
- Trustworthy: Enterprise-grade security and reliability
- Innovative: Modern tech, open-source, future-thinking
- Approachable: Human, not corporate jargon
- Indian: Culturally relevant, locally built
- Transparent: Clear pricing, open development

SECONDARY TRAITS:
- Efficient: Saves time, reduces complexity
- Empowering: Enables business growth
- Community-driven: Collaborative, supportive
- Sustainable: Built for long-term success

VISUAL METAPHORS:
- The Bridge: Connecting traditional accounting to modern technology
- The Local Store: Accessible, trusted neighborhood solution
- The Open Book: Transparency, clarity, no hidden information
- The Rising Graph: Growth, success, upward trajectory

DESIGN PRINCIPLES:

1. CLARITY OVER CLEVERNESS
   Every element should communicate its purpose instantly. No mystery meat 
   navigation, no abstract icons without labels, no hidden functionality.

2. CONFIDENCE OVER AGGRESSION  
   Bold but not boastful. Present features as facts, not exaggerations.
   Use data and testimonials as proof points.

3. WARMTH OVER RIGIDITY
   Professional doesn't mean cold. Use human imagery, conversational copy,
   and Indian cultural elements subtly integrated.

4. PURPOSE OVER DECORATION
   Every visual element must serve a function. No gratuitous gradients, 
   shadows, or animations that don't enhance usability.

5. CONSISTENCY OVER VARIETY
   Establish a strict design system and stick to it. Consistency builds 
   trust and reduces cognitive load.

2. COLOR PALETTE & COLOR PSYCHOLOGY
================================================================================

PRIMARY BRAND COLORS:

VERSOLL BLUE (#1E40AF)
- Role: Primary brand color, CTAs, key interactive elements
- Psychology: Trust, stability, professionalism, technology
- Usage: Navigation, buttons, links, logo, accents
- Contrast: White text passes WCAG AAA (7.2:1 ratio)
- Indian Context: Corporate trust, reminiscent of trusted banking institutions

VERSOLL SAFFRON (#EA580C)  
- Role: Secondary brand color, highlights, energy
- Psychology: Energy, enthusiasm, creativity, Indian identity
- Usage: Hover states, active elements, highlights, accent borders
- Contrast: White text passes WCAG AA (4.3:1 ratio)
- Indian Context: National color, evokes trust and cultural connection
- Balance: Use sparingly (5-10% of page) to avoid overwhelming

VERSOLL GREEN (#059669)
- Role: Success states, positive metrics, growth indicators
- Psychology: Success, growth, stability, financial wellness
- Usage: Success messages, positive stats, "save" actions, completed states
- Contrast: White text passes WCAG AA (4.5:1 ratio)
- Context: Financial growth, accounting reconciliation

NEUTRAL PALETTE:

DARK SLATE (#0F172A)
- Role: Primary text, headings, strong contrast elements
- Usage: All headings (H1-H6), body text, navigation text
- Contrast: White background at 15.3:1 (WCAG AAA)
- Psychology: Authority, professionalism, sophistication

SLATE GRAY (#334155)
- Role: Secondary text, labels, metadata
- Usage: Subheadings, captions, form labels, secondary info
- Contrast: 8.6:1 on white (WCAG AAA)

GRAY MEDIUM (#64748B)
- Role: Tertiary text, placeholders, disabled states
- Usage: Placeholder text, disabled buttons, inactive tabs
- Contrast: 4.6:1 on white (WCAG AA)

LIGHT GRAY (#F1F5F9)
- Role: Section backgrounds, subtle separators, card backgrounds
- Usage: Page section backgrounds, table rows, input fields

WHITE (#FFFFFF)

COLOR USAGE RULES:

1. MAXIMUM 4 COLORS PER PAGE
   Typically: Dark Slate (text), White (bg), Versoll Blue (accents), 
   one semantic color for feedback

2. SAFFRON USE LIMITATION
   Never more than 10% of viewport at any time
   Reserve for high-importance CTAs only
   Maximum 2 saffron CTAs per page (only 1 can be primary)

3. ACCESSIBILITY REQUIREMENTS
   All text must pass WCAG AA minimum (4.5:1)
   Interactive elements must pass WCAG AAA when possible (7:1)
   Never rely on color alone for meaning (use icons + text)

4. DARK MODE ADAPTATION
   When dark mode is implemented (Phase 2):
   - Primary: #3B82F6 (light blue)
   - Background: #0F172A (dark slate)
   - Text: #F1F5F9 (light gray)
   - Cards: #1E293B (slightly lighter)

3. TYPOGRAPHY SYSTEM
================================================================================

TYPEFACE STRATEGY:

PRIMARY: Inter (by Rasmus Andersson)
- Rationale: Designed specifically for UI, excellent readability, 
  extensive weight range, open-source, optimized for screens
- Usage: All headings, body text, UI elements
- Load: Google Fonts or self-hosted WOFF2 (recommended)
- Performance: <20kb with modern compression

SECONDARY: Noto Sans Devanagari
- Rationale: For Hindi/Marathi/Gujarati content
- Usage: Regional language versions (Phase 2)

ACCENT: JetBrains Mono
- Rationale: Monospace for code, numbers, data tables
- Usage: Invoice numbers, amounts, API keys, technical data

TYPE SCALE:

DESKTOP SCALE (Base: 16px):
H1: 48px / 1.2 line-height / -0.02em tracking
   Usage: Page titles, hero headlines
   Weight: 700 (Bold)
   Mobile: 36px

H2: 36px / 1.25 line-height / -0.01em tracking  
   Usage: Section titles
   Weight: 600 (SemiBold)
   Mobile: 28px

H3: 30px / 1.3 line-height / 0em tracking
   Usage: Subsection titles, card titles
   Weight: 600 (SemiBold)
   Mobile: 24px

H4: 24px / 1.35 line-height / 0em tracking
   Usage: Feature titles, form section headers
   Weight: 500 (Medium)
   Mobile: 20px

H5: 20px / 1.4 line-height / 0em tracking
   Usage: Small headings, metadata labels
   Weight: 500 (Medium)
   Mobile: 18px

H6: 16px / 1.5 line-height / 0em tracking
   Usage: Labels, captions
   Weight: 500 (Medium)
   Mobile: 16px

Body Large: 18px / 1.6 line-height
   Usage: Hero descriptions, intro paragraphs
   Weight: 400 (Regular)
   Mobile: 16px

Body Regular: 16px / 1.5 line-height
   Usage: Main content, descriptions, form labels
   Weight: 400 (Regular)
   Mobile: 15px

Body Small: 14px / 1.4 line-height
   Usage: Captions, footnotes, metadata
   Weight: 400 (Regular)
   Mobile: 14px

Caption: 12px / 1.3 line-height
   Usage: Fine print, legal text, disclaimers
   Weight: 400 (Regular)
   Mobile: 12px

BUTTON TEXT: 16px / 1.2 (all caps, tracked)
   Weight: 500 (Medium)
   Letter-spacing: 0.05em
   Mobile: 14px

TYPOGRAPHY RULES:

1. LINE LENGTH LIMIT
   Maximum 75 characters per line (body text)
   Optimal 55-65 characters for readability
   Use CSS max-width: 65ch for containers

2. VERTICAL RHYTHM
   Baseline grid: 4px
   All spacing in multiples of 4px (8px, 12px, 16px, 24px, 32px, etc.)

3. FONT WEIGHTS ALLOWED
   Only 4 weights: 400 (Regular), 500 (Medium), 600 (SemiBold), 700 (Bold)
   Avoid 300 (Light) - readability issues on Windows
   Avoid 800-900 (ExtraBold) - too aggressive for brand

4. EMPHASIS TECHNIQUE
   Use font-weight: 600 instead of italic for emphasis (Indian languages don't 
   emphasize well with italics)
   Reserve italics for quotes and citations only

5. NUMBER FORMATTING
   Use tabular numbers (font-variant-numeric: tabular-nums) for financial data
   Improves readability in tables and invoices

7. ANIMATION & MICRO-INTERACTIONS
================================================================================

ANIMATION PHILOSOPHY:

PRINCIPLE: "Purposeful Motion with Indian Sensibility"
- Animations should explain, guide, or delight
- Never distract from core tasks
- Respect user's time (be quick)
- Reflect brand personality (confident, not flashy)

TIMING SYSTEM:

FAST (150ms)
- Hover states
- Button press feedback
- Form field focus
- Micro-interactions
Easing: ease-out

MEDIUM (300ms)
- Page transitions
- Modal opens/closes
- Accordion expands
- Tab switches
Easing: ease-in-out

SLOW (600ms)
- Major section reveals
- Chart animations
- Hero element entry
- Complex state changes
Easing: Custom cubic-bezier(0.4, 0, 0.2, 1)

SPECIFIC ANIMATIONS:

1. HERO SECTION ENTRY (Single Load)
- Sequence: Title → Subtitle → Buttons → Hero Image
- Delay staggers: 150ms between each
- Transform: Fade in + slide up 20px
- Total duration: 1200ms from page load
- Only plays once (prefers-reduced-motion respect)

2. BUTTON HOVER (Repeating)
- Transform: translateY(-2px)
- Shadow: 0 4px 12px rgba(30, 64, 175, 0.3)
- Duration: 150ms ease-out
- Active: scale(0.98) + shadow removal

3. CARD HOVER (Desktop only)
- Transform: translateY(-4px)
- Shadow: 0 8px 25px rgba(0,0,0,0.15)
- Duration: 200ms ease-out
- Scale: 1.01 (subtle)

4. FEATURE ICON ANIMATION
- Icon floats 8px up/down in 3s loop
- Opacity pulses 0.9 to 1.0
- Purpose: Subtle attention without being annoying
- Pause on hover

5. COUNTDOWN TIMER (CRM Page)
- Numbers flip animation (300ms)
- Color: Transitions from Versoll Blue to Saffron in final day
- Pulsing effect when under 24 hours
- Updates every second

6. PROGRESS BARS
- Width animation from 0% to target %
- Duration: 800ms delay 200ms
- Easing: easeOutQuart
- Color: Gradient from Blue to Green
- Percentage text counts up in sync

7. FORM VALIDATION
- Error state: Input shakes (400ms), border turns red (200ms)
- Success state: Checkmark slides in from left (300ms)
- Real-time: Inline validation after 500ms debounce

8. PAGE TRANSITIONS
- Fade out current: 150ms
- Loading spinner: Versoll logo animation
- Fade in new: 300ms
- Total: <500ms perceived speed

9. TESTIMONIAL CARDS
- Entry: Staggered reveal on scroll
- Each card slides up 30px + fade in
- 150ms stagger, 600ms total
- Pause when 50% visible (IntersectionObserver)

10. COMPARISON TABLE REVEAL
- Desktop: Each row slides in from left, staggered
- Mobile: Accordion expand animation (height auto)
- Duration: 300ms per row
- Easing: ease-out

REDUCED MOTION RESPECT:
```
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

9. ACCESSIBILITY STANDARDS
================================================================================

WCAG 2.1 AA COMPLIANCE (BASELINE):

1. COLOR CONTRAST
- Text on background: 4.5:1 minimum (AA)
- Large text (18+px): 3:1 minimum (AA)
- Interactive elements: 4.5:1 minimum
- Icons: 3:1 minimum
- Tested with: WebAIM Contrast Checker

2. KEYBOARD NAVIGATION
- All interactive elements reachable via Tab
- Clear focus indicators (2px solid #EA580C outline)
- Logical tab order (DOM order)
- Skip links: "Skip to main content" (visible on focus)

3. SCREEN READER SUPPORT
- Semantic HTML5 elements (nav, main, article, section)
- ARIA labels where needed
- Alt text for all images (descriptive)
- Form labels properly associated
- Error messages announced (aria-live)

4. VISIBLE FOCUS INDICATOR
```
*:focus {
  outline: 2px solid #EA580C;
  outline-offset: 2px;
  border-radius: 2px;
}
```

5. HEADING HIERARCHY
- H1: Page title (only one per page)
- H2: Major sections
- H3: Subsections
- No skipping levels (H2→H4 is not allowed)

6. LINK TEXT
- Descriptive text (not "click here" or "learn more")
- Example: "Read the GST compliance guide" instead of "Learn more"
- Unique link text (avoid duplicates)

7. FORM ACCESSIBILITY
- Labels for every input
- Fieldset/legend for groups
- Error messages linked with aria-describedby
- Success messages announced
- Required fields marked (asterisk + aria-required)

8. IMAGES & MEDIA
- Alt text: Describe function, not appearance
- Decorative images: alt=""
- Complex charts: Longdesc or linked description
- Videos: Captions required, transcript available
